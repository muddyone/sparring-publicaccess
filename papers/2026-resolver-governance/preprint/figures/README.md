## fig1 — the redirect spine (§5.1)

A Cleveland dot plot of judge-vs-human agreement (Spearman ρ) per rubric criterion,
**grouped by whether the process evidence stayed visible after the format-symmetric
blinding** (§5.1). Agreement held where the blind left evidence visible (C1, ρ=0.77) and
collapsed where it stripped it (C2 0.32, C3 −0.50). **Unchanged** — reads the rater study,
not v3.

Integrity choices baked in (do not "fix" these — they are the point):
- ρ axis is honest `[-1, +1]`; no truncation.
- Pass/fail thresholds are drawn **per row** (tiers differ: high-stakes 0.70/0.40,
  inference-heavy 0.50/0.20). A single global band would misrepresent the inference-heavy criteria.
- Rows ordered by the **independent variable** (visibility), never sorted by ρ (circular).
- **C5a** has no ρ (flat 6/7) → shown as an explicit INDETERMINATE row, never faked.
- The n=4-per-ρ small-n caveat, the 6-rater reference, and "directional, not clean (C4 also
  fails)" are printed on the figure.

## catch-rate-2x2 — the Generator×Challenger×Audit grid (§6.3)

A heatmap (values printed) of the full 2×2×2 factorial: 5 review conditions × 4 generator×audit
pairings. The only catch-rate cut that preserves the load-bearing hedge — a challenge recovers
78–94% of flaws in all cells, but whether a *separate* challenger beats self-review is
**generator-dependent** (large for Claude-produced work, marginal for GPT-5.2, and the Claude
blind-spot persists under a GPT audit → real, not same-vendor favoritism). The produce-and-ship
row (0) is a definitional anchor, not a measured cell. Caveats on the figure: n≈30–32 flaws/cell,
model-established ground truth, blind-coded by one held-constant coder, directional, no sig. test.

## fig2-grounding-v3 — the v3 grounding dot plot (§7.2)

**Re-cut** from the earlier 2-D "theater map" scatter (superseded — see the generator docstring).
A Cleveland dot plot of grounding rate on a common 0–100 scale: B 58% / T 83% / S 96%. Position
encoding for the one number that carries the argument; precision is a secondary annotation (it is
100/100 on the load-bearing S-vs-T contrast, so it earns no axis of its own). The honest story:
the **durable** win is S over a raw pass (S−B = 38 pts, at no precision cost); the edge over a
**competent** skeptic is **marginal** (S−T ≈ 13 pts, n=12, no sig. test). Caveats on figure:
grounding machine-verified (existence model-free, correct-application cross-vendor); precision/
legitimacy LLM-coded, expert-pending; directional, n=12 cases / 99 concerns.

*Earn-the-ink note:* the §7.3 by-family concentration is **deliberately left as the in-text table,
not charted** — each family rests on only 3 cases, and §7.3 itself says the magnitudes are too
few to trust (the contradiction "reversal" is as plausibly noise as signal). A figure of those
magnitudes would over-dignify noise — the cardinal-rule failure this discipline exists to prevent.

## fig4-coverage-v3 — the verifier's coverage in v3 (§8 / §10.2)

**Replaces** the superseded `fig3-verifier-ceiling`. A part-to-whole bar of all **99** v3
concerns by how their grounding is resolved: in-briefing explicit 70 + mapper-recovered 16
(= 86), external registry 6, no-artifact 7 — so **92 of 99 are machine-checkable**. Headlines the
*closed* ceiling (v2 ~7%-only → v3 92/99, a data-loss artifact fixed by persisting briefings) and
that the residual limit is now **legitimacy**, not coverage (legitimacy → domain-expert audit).
Every number is read from the frozen `v3-run/coverage.json` (derived by that pilot's committed
`derive_coverage.py` from the canonical case-file codings); asserts guard the part-to-whole and
breakdown sums. Caveats on figure: existence model-free, correct-application cross-vendor,
directional/synthetic, n=99/12.

## Regenerating

```bash
python3 figures/fig1-redirect-spine.py        # stdlib only
python3 figures/fig2-grounding-v3.py          # needs matplotlib
python3 figures/fig4-coverage-v3.py           # needs matplotlib
# Fig 2 (catch-rate-2x2) regenerates from the pilot dir, then copy the svg here:
( cd ../../../catch-rate-v3-2026-06-28 && python3 figures/catch-rate-2x2.py )
cp ../../../catch-rate-v3-2026-06-28/figures/catch-rate-2x2.svg figures/catch-rate-2x2.svg
```

All generators read frozen JSON and echo the exact values they plotted (so a reviewer can diff
figure against source); fig1 also self-checks well-formed XML. The matplotlib generators set a
fixed `svg.hashsalt` and write `metadata={"Date": None}`, so re-runs are **byte-identical**.

## Embedding in the build

- **HTML build** (canonical clean artifact): SVG embeds natively. `![caption](figures/<name>.svg)`.
- **PDF build** (xelatex, best-effort): `rsvg-convert` (`librsvg2-bin`, installed here) lets pandoc
  embed SVG; on a machine without it the PDF step skips the figure.
- **Preview without a build:** `rsvg-convert -w 1100 figures/<name>.svg -o preview.png`.
