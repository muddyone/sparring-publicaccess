## Recommendation

Adopt Path B (language-first) — students writing runnable Python from week 1 — plus one bounded "concept overlay" authored once by the four full-time faculty: a recurring one-page labeled sidebar that names the underlying idea behind each week's code (decomposition, control flow, data organization), and 3–4 reusable "rewrite this in plain steps" lab prompts. Path B runs on the materials NCCC has used since 2018, so adjuncts can deliver it cold; the overlay is a fixed artifact set (a few faculty-days of work), not a teaching skill an adjunct must re-perform live each week. Do not adopt the full per-week dual-frame hybrid (undeliverable across this staffing pattern) or pure Path A.

## Reasoning chain

Ordering barely moves completion. Evidence 2 found completion varied 65–78% across course *designs* but 58–85% across *languages* — and the language is already fixed at Python. So the design target is not chasing a completion lift from ordering (there isn't one); it is serving this specific student body and the course's goals on terms the college can actually deliver.

Path B fits the majority. Evidence 3 shows adult learners — 57% of this body — are motivated by early concrete competence ("by week 3 you've run a working program"), exactly what language-first onset provides; pure Path A's five weeks before any running code is the demotivator for that majority. And per Evidence 2, the 62% who do not continue lose nothing on finishing from this choice.

Deliverability is decisive, and it favors Path B. The staffing pattern (4 full-time + 6–8 adjuncts across ~8 sections) and the June 30 build deadline mean the modal section is adjunct-led with no shared prep. Path B leans on existing 2018-onward materials; the overlay is the only net-new authoring, small and front-loaded onto full-time faculty.

The first-generation (31%) and second-language (20%) students trail ~12 points on write-from-scratch tasks (Evidence 4), partly from an unspoken sense of what a good solution looks like — which the overlay's "name the standard" prompts address. But Evidence 4 failed to replicate in Lewis 2022, so the overlay hedges this rather than betting the design on it.

## Concerns / risks

- Path B does not secure the transfer edge in Evidence 5 (concepts-first scored d ≈ 0.18–0.22 higher — about 3–4 points — but only for the ~38% who continue *and* reach data structures within 18 months). The concept overlay is an unproven partial proxy, not a substitute. This is the real, owned cost of not choosing Path A.
- It knowingly trades away Evidence 1's cleaner cognitive-load argument for concepts-first, in exchange for deliverability.
- Completion is not moved by any path choice (Evidence 2); the sub-75% completion concern is a preparation problem the curriculum cannot fix, and this recommendation does not promise to.
- The equity benefit rests on contested evidence (Evidence 4 / Lewis 2022 non-replication); the overlay hedges but does not resolve it.

## Confidence

Medium. The choice is grounded in the completion-neutrality of ordering (Evidence 2), the adult-learner majority's need for early competence (Evidence 3), and — decisively — what is actually deliverable under the stated staffing pattern and deadline. The main residual uncertainty is the conceded transfer-edge trade-off (Evidence 5): real, but accruing to a minority subgroup under a condition the course cannot control.
